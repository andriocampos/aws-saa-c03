## Dica da semana
IAM é transversal — ao estudar qualquer outro serviço, sempre pergunte:
"Quem tem permissão para fazer isso? Via qual role/policy?"
Foco especial em: AssumeRole, Trust Policy e a lógica Allow/Deny.
